# Localization

Cognigy.AI offers a robust localization feature for easy customization and content reuse. You can configure multiple Locales and add localized content to your Flow. Alternatively, you can use [machine translation to automatically translate content](auto-translation.mdx). If one Locale doesn't have configured content, Cognigy.AI automatically falls back to another Locale.

## Restrictions

<Tabs>
  <TabItem value="tab1" label="GUI" default>
    You can add, edit, and delete Locales in **Manage > Localization**. Also, you can copy their Reference ID.

  </TabItem>
  <TabItem value="tab2" label="API">
    You can view, create, edit, and delete Locales using the [Cognigy.AI API](https://api-dev.cognigy.ai/openapi#tag--Locales-v2.0).

  </TabItem>
</Tabs>
- You can't change the primary Locale after you've selected it.

## How to Use

### Localize a Flow

You can localize [Nodes](../nodes/overview.mdx) and [Intents](../../empower/nlu/intents/ml-intents.mdx) in the [Flow editor](../flows/editor.mdx) after selecting a Locale in the top bar.
